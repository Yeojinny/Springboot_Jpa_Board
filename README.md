1. 소개

순수JDBC를 사용하여 JAVA로 Mysql과 연동하는 코드를 구현하였을 때, 메서드마다 jdbc 드라이버와 연결하는 코드를 작성하고
데이터에 따라 매번 다른 쿼리를 작성하는 부분에 있어서 반복이 많고 코드가 상당히 길다고 생각하였습니다.
이러한 반복없이 좀 다른 방법으로 데이터베이스와 연동하는 방법은 없는지 궁금하였고, 
최근에는 Spring boot에서 Jpa를 사용하여 연동하는 방식을 많이 사용한다는 것을 알게 되었습니다.
따라서, 수업시간외에 해당 기술을 통해 간단한 게시판을 구현하는 부분을 따라하며 공부해보았습니다.
활용한 블로그 : https://victorydntmd.tistory.com/325

2. 기능

가장 기본적인 CRUD
- 게시물을 작성하여 데이터베이스에 저장
- 메인 페이지에서 글쓰기, 저장된 게시물 리스트 조회, 검색
- 게시글 상세 조회, 수정, 삭제

3. 개발 환경

- 운영체제 : 우분투 리눅스
- IDE : Intellij IDEA
- Back-end : Springframework 2.3.9 release(라이브러리 JPA, Spring web, Thymeleaf ...)
- Database : Mysql

간단한 게시판 만들기를 학습하며, Spring과 JPA를 통해 기존의 반복 코드를 확줄일 수 있고 기본적인 SQL도 JPA가 직접 만들어서
실행해준다는 부분에 놀랐습니다. 아직 해당프레임워크에 대한 이해가 부족하지만 계속해서 공부해나가야겠다고 생각했습니다.
현재까지는 코드를 따라가며 학습해보았지만 추후에는 개인적으로 기능들을 추가 구현해나가며 발전시켜나갈 예정입니다.


웹 개발관련하여 프론트엔드쪽으로는 접해보지 못했지만, 아래와 같은 내용을 공부하며 구현한 경험이 있습니다.
- Node.js 기반으로 간단한 서버 구현 및 db연동
- 간단한 채팅 프로그램 코드를 바탕으로 소켓 프로그래밍 학습
- jdbc를 통한 db연동 및 crud 실행 / spring boot jpa 활용한 db연동

실질적인 프로젝트 경험은 없지만 최대한 학습했던 것을 활용하고,
부족한 부분에 대해서는 지속적인 공부해나감으로써 빠르게 역량을 키워나갈 자신이 있습니다.
